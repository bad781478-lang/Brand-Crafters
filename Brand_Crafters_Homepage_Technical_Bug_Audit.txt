BUG-01 - Broken anchor: scroll cue dead link
Severity
High
File target
Search for scroll-down-container or signal-section in src/
Area
Services scroll cue / cinematic grid anchor

Problem
A services-section scroll cue links to #signal-section, but no rendered DOM element has id="signal-section". Clicking the cue does nothing.
Required Fix
Change the scroll cue href from #signal-section to #media-system-section.
Reason: The cinematic grid section already renders id="media-system-section", which is the correct next section in the scroll narrative.
Grep to Find
grep -rn "signal-section" ./src/
Fix Steps
	•	Find the scroll cue anchor tag with class scroll-down-container.
	•	Change href="#signal-section" to href="#media-system-section".
	•	Add the required implementation comment near the change: // BUG-FIX: BUG-01.
Verify
grep -n "signal-section\|media-system-section" ./src/**/*.jsx
signal-section should return 0 results.
media-system-section should exist in one file.
BUG-02 - Non-functional service cards not clickable
Severity
High
File target
Search for OurServicesSection or services-section in src/
Area
OurServicesSection / services-section cards

Problem
The four service cards show cursor:pointer and a View Details CTA bar, but the card wrapper is a non-interactive div with no click, keyboard, or navigation behavior.
Required Fix
Make each card operate as a keyboard-accessible button-style card that routes to the correct service anchor.
Reason: The existing visual treatment promises interaction; the implementation must preserve that appearance while making the interaction real.
Grep to Find
grep -rn "View Details\|cursor-pointer" ./src/ --include="*.jsx"
Fix Steps
	•	Locate the four service card outer divs with classes containing group sticky or group lg:relative and cursor-pointer.
	•	Add role="button", tabIndex={0}, an onClick route, an aria-label, and an Enter/Space onKeyDown handler to each card.
	•	Map Brand Identity to /services#brand-identity.
	•	Map Websites & Apps to /services#websites-apps.
	•	Map Creative Production to /services#creative-production.
	•	Map Marketing & Visibility to /services#marketing.
	•	If React Router is already used in that file, prefer useNavigate() and navigate('/services#...') over window.location.href.
	•	Change the inner View Details element from div to span so the button-role parent does not contain nested interactive content.
	•	Add the required implementation comment near each card change: // BUG-FIX: BUG-02.
Implementation Pattern
<div
  className="group sticky lg:relative ... cursor-pointer origin-top"
  onClick={() => window.location.href = '/services#brand-identity'}
  role="button"
  tabIndex={0}
  aria-label="View Brand Identity service details"
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      window.location.href = '/services#brand-identity';
    }
  }}
>
Verify
Each card has role, tabIndex, onClick, onKeyDown, and aria-label.
No nested button or anchor exists inside the card div.

BUG-04 - Accessibility: FAQ not keyboard navigable
Severity
High
File target
Search for FAQ or faq-section in src/
Area
FAQ accordion

Problem
FAQ question rows are plain div elements with onClick handlers and focus:outline-none. Keyboard users cannot tab to the rows, and screen readers cannot identify them as interactive controls.
Required Fix
Convert each question row to a real button with accessible expanded-state wiring and visible keyboard focus.
Reason: This resolves the WCAG 2.1 AA keyboard access failure without changing the visual design.
Grep to Find
grep -rn "faq.*click\|onClick.*faq\|FAQ\|accordion" ./src/ --include="*.jsx" -l
Fix Steps
	•	Find every FAQ question row with flex items-center justify-between, cursor-pointer, and onClick.
	•	Change each question row from div to button type="button".
	•	Add id={`faq-question-${index}`} to each question button.
	•	Add aria-expanded={openIndex === index} and aria-controls={`faq-answer-${index}`}.
	•	On the expanded answer div, add id={`faq-answer-${index}`}, role="region", and aria-labelledby={`faq-question-${index}`}.
	•	Remove focus:outline-none and replace it with the focus-visible ring pattern.
	•	Keep the h3 inside the button if present; button > h3 is valid HTML.
	•	Add the required implementation comment near the FAQ change: // BUG-FIX: BUG-04.
Implementation Pattern
<button
  type="button"
  className="flex items-center justify-between w-full py-5 ... cursor-pointer focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-white/40 focus-visible:ring-offset-2 focus-visible:ring-offset-transparent text-left"
  id={`faq-question-${index}`}
  aria-expanded={openIndex === index}
  aria-controls={`faq-answer-${index}`}
  onClick={() => toggleFaq(index)}
>
Verify
Tab through the page in browser dev tools.
All seven FAQ items are reachable via Tab.
Enter and Space toggle open/closed state.
aria-expanded updates correctly.

BUG-06 - Performance: video preload strategy wrong
Severity
Medium
File target
Search for preload="auto" and preload in JSX files
Area
BusinessTrackSection and StickyCinematicGrid

Problem
The cinematic grid videos use preload="auto", forcing nine videos to start downloading on page load even before users scroll near the section.
Required Fix
Keep hero video preload="auto", but set cinematic grid videos to preload="none" until the grid section enters the viewport.
Reason: The hero video is likely LCP and should load immediately; below-the-fold grid videos should wait until the ScrollTrigger hook is active.
Grep to Find
grep -rn 'preload="auto"' ./src/
grep -rn "preload" ./src/ --include="*.jsx"
Fix Steps
	•	In StickyCinematicGrid, change video preload="auto" to preload="none".
	•	Inside the existing GSAP ScrollTrigger hook for the grid section, query gridRef.current for video elements when the section enters the viewport.
	•	Set each grid video preload to auto inside that hook.
	•	Do not change the hero video preload behavior.
	•	Add the required implementation comment near the grid preload logic: // BUG-FIX: BUG-06.
Implementation Pattern
const videos = gridRef.current.querySelectorAll('video');
videos.forEach((v) => {
  v.preload = 'auto';
});
Verify
grep -rn 'preload="auto"' ./src/
Only HeroSection should contain preload="auto".
All other video elements should use preload="none" initially.
BUG-07 - Missing scroll target: FAQ top margin
Severity
Low
File target
FAQ section component
Area
FAQ section deep link target

Problem
The FAQ section uses -mt-[100vh] to create an overlap effect, but direct links to #faq-section scroll to the wrong position because the negative margin changes the target position.
Required Fix
Create a zero-height invisible anchor above the negatively-margined section, then rename the visual section id to faq-section-panel.
Reason: Deep links can land correctly while preserving the existing overlap effect.
Grep to Find
grep -rn "faq-section\|-mt-\[100vh\]" ./src/ --include="*.jsx"
Fix Steps
	•	Find the FAQ section element with id="faq-section".
	•	Insert a zero-height hidden anchor div with id="faq-section" immediately above the section.
	•	Change the existing section id to faq-section-panel.
	•	Update footer FAQ links, if present, so they still point to #faq-section.
	•	Add the required implementation comment near the anchor: // BUG-FIX: BUG-07.
Implementation Pattern
<div
  id="faq-section"
  style={{ position: 'relative', top: 0, height: 0, visibility: 'hidden', pointerEvents: 'none' }}
  aria-hidden="true"
/>
<section
  id="faq-section-panel"
  className="... -mt-[100vh]"
>
Verify
grep -rn "faq-section" ./src/
The scroll anchor div should appear before the section element.
